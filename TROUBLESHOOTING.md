# 🔧 Вирішення проблеми 400 при налаштуванні Apps Script

## ❌ Проблема

При спробі відкрити **Extensions → Apps Script** в Google Sheets виникає помилка 400.

## 🔍 Можливі причини

1. **Немає прав на редагування таблиці** - ви не є власником
2. **Apps Script API вимкнено** в Google Cloud проекті
3. **Google Workspace обмеження** - адміністратор заборонив Apps Script
4. **Таблиця в режимі перегляду** - потрібні права Editor

## ✅ Рішення

### Варіант 1: Створити власну копію таблиці (Рекомендований)

```bash
# 1. Відкрийте вашу Google Sheets
# 2. File → Make a copy
# 3. Дайте назву: "UA Yogis 2026 - Dashboard Copy"
# 4. Тепер спробуйте Extensions → Apps Script
```

**Після створення копії:**
1. Скопіюйте новий SHEET_ID з URL
2. Оновіть `.env`:
   ```bash
   INCOME_SHEET_ID=новий-sheet-id-тут
   ```
3. Запустіть:
   ```bash
   npm run setup-webhook
   ```

### Варіант 2: Standalone Apps Script (Обхід проблеми)

Створити Apps Script **окремо** від таблиці:

```bash
npm run setup-standalone
```

**Що робить:**
1. ✅ Відкриває https://script.google.com/create
2. ✅ Копіює готовий код в буфер обміну
3. ✅ Створює файл `webhook-standalone.gs`

**Налаштування:**
1. Вставте код (Ctrl+V)
2. Збережіть (Ctrl+S)
3. Налаштуйте тригер:
   - Значок ⏰
   - Function: `syncData`
   - Event: Time-driven
   - Every: 1 minute

**Переваги standalone:**
- ✅ Не потрібні права на таблицю
- ✅ Працює з будь-якою таблицею (навіть чужою)
- ✅ Обходить помилку 400

**Недоліки:**
- ⏱️ Оновлення кожну хвилину (не миттєво)
- 📊 Використовує Time-driven тригер замість onEdit

### Варіант 3: Без webhook (Тільки polling)

Якщо webhook не критичний, можна працювати **тільки з auto-polling**:

```bash
# У .env НЕ потрібен WEBHOOK_SECRET
# Сайт буде оновлюватися кожні 30 секунд автоматично
```

**Як це працює:**
```
Dashboard.tsx → useEffect → кожні 30 сек → /api/data → Google Sheets API → Оновлення
```

**Переваги:**
- ✅ Не потрібен Apps Script взагалі
- ✅ Працює одразу
- ✅ Немає помилок 400

**Недоліки:**
- ⏱️ Затримка до 30 секунд
- 📡 Більше API запитів

### Варіант 4: Увімкнути Apps Script API

Якщо у вас є доступ до Google Cloud Console:

1. Зайдіть на https://console.cloud.google.com/
2. Виберіть ваш проект (або створіть новий)
3. APIs & Services → Library
4. Знайдіть "Google Apps Script API"
5. Натисніть Enable
6. Спробуйте знову Extensions → Apps Script

## 🚀 Швидкий старт (без помилок)

### Якщо помилка 400 - виконайте це:

```bash
# 1. Створіть копію таблиці в Google Sheets
#    File → Make a copy

# 2. Скопіюйте новий SHEET_ID з URL і оновіть .env

# 3. Запустіть автоматичне налаштування
npm run setup-webhook

# 4. Слідуйте інструкціям (30 секунд)

# ✅ Готово! Помилки немає, webhook працює
```

### Якщо не хочете возитися зі скриптами:

```bash
# Просто запустіть dev сервер
npm run dev

# Сайт працює з auto-polling (оновлення кожні 30 сек)
# Webhook не обов'язковий!
```

## 💡 Рекомендації

**Для production:**
- ✅ Використовуйте автоматичний polling (уже працює)
- ✅ Додайте webhook для миттєвих оновлень (опціонально)
- ✅ Деплойте на Vercel

**Для development:**
- ✅ Локальний JSON файл (`data/financial-data-2026.json`)
- ✅ `npm run fetch-data` для ручного оновлення
- ✅ `npm run dev` для розробки

## 📞 Все ще не працює?

Перевірте:
1. ✅ Чи правильний GOOGLE_API_KEY в `.env`
2. ✅ Чи таблиця публічна або API ключ має права
3. ✅ Чи INCOME_SHEET_ID правильний
4. ✅ Чи INCOME_SHEET_NAME правильний

Запустіть тест:
```bash
npm run fetch-data
```

Якщо дані завантажуються - все працює! Webhook не обов'язковий. 🎉

---

**Підсумок:** Помилка 400 - не критична. Сайт працює з auto-polling без жодних скриптів!
